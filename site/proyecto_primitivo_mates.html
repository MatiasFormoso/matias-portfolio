Primitivo Mates — e-commerce con Mercado Pago

Tienda online de mates con checkout integrado, webhooks de pago y gestión de órdenes. El negocio se discontinuó; el deploy técnico demuestra la integración completa.

Next.jsTypeScriptMercado Pago WebhooksSupabaseE-commerce

Contexto

Primitivo Mates fue un emprendimiento de e-commerce para venta de mates artesanales. El desafío técnico principal fue integrar Mercado Pago de punta a punta: preferencias de pago, confirmación vía webhooks y actualización de stock/órdenes.

Estado del proyecto

  • ▶ Negocio discontinuado — no es un e-commerce activo.
  • ▶ La integración técnica (MP + webhooks) quedó implementada y desplegada.
  • ▶ Experiencia aplicable a roles de integraciones y pagos.

Qué hice

  • ▶ Integración completa de Mercado Pago (SDK + API de preferencias).
  • ▶ Endpoint de webhooks con validación HMAC para confirmación de pagos.
  • ▶ Flujo de órdenes: creación, estados (pendiente/aprobado/rechazado) y notificaciones.
  • ▶ Gestión de stock en Supabase (PostgreSQL) con actualización post-pago.
  • ▶ Frontend Next.js con carrito, checkout y páginas de producto.

Decisiones técnicas

  • Webhooks idempotentes: reintentos de MP no duplican órdenes ni descuentan stock dos veces.
  • Validación HMAC: rechazo de payloads no firmados por Mercado Pago.
  • Estados explícitos: máquina de estados simple para trazabilidad de cada transacción.
  • Supabase como backend liviano: sin servidor propio para MVP rápido.
